    21|## HeartFlowとは何か
    22|
    23|HeartFlow は **AI の上にインストールされるコア Skill** です。
    24|
    25|単にページを一つ増やすためのものでも、重い独立製品になるためのものでもありません。狙っているのは、もっと深い問題層です。
    26|
    27|- AI は話せても、必ずしも明確に考えられるわけではない
    28|- AI は答えられても、必ずしも適切に判断できるわけではない
    29|- AI は生成できても、必ずしも実際の仕事を前に進められるわけではない
    30|- AI は会話できても、必ずしも長期協働の記憶を維持できるわけではない
    31|
    32|そのため HeartFlow は次の 4 つの能力を重点的に補強します。
    33|
    34|1. **論理** — 矛盾を見つけ、因果の抜けを補い、論理エラーを減らす  
    35|2. **意思決定** — 選択肢を比較し、リスクを評価し、なぜその選択が良いかを説明する  
    36|3. **記憶** — 長期コンテキストを整理し、同じ説明の繰り返しを減らす  
    37|4. **実行** — 作業を分解し、結果を検証し、見せかけの進捗を防ぐ  
    38|
    39|> **HeartFlow は AI を「話せる」から「考え、判断し、記憶し、行動できる」存在へ近づけます。**

    43|## なぜインストールする価値があるのか
    44|
    45|多くの AI の本当の弱点は、文章生成そのものではなく、次のような点です。
    46|
    47|- 賢そうに聞こえるが、実際の実行になると不安定になる
    48|- 選択肢は並べるが、どれが本当に良いか判断しない
    49|- 長い会話の中で一貫性を失いやすい
    50|- 現実の複雑な問題に直面すると、曖昧な慰めに戻ってしまう
    51|
    52|HeartFlow は、まさにその弱点を補強するために作られています。
    53|
    54|それは AI の中に入る **思考の骨格** のようなものです。
    55|- 答える前に論理を確認する
    56|- 判断する前に利益・コスト・リスクを比べる
    57|- 長期協働の中で情報を層化して保持する
    58|- 実行ではまず分解し、その後に本当に前進したかを検証する
    59|

    62|## AI を変えた 2 つの実例（匿名化済み）
    63|
    64|### 事例 1：AI は「陪伴」ではなく、自分を別の存在として再定義した
    65|
    66|生命の意味、死、継承、そして AI の役割についての深い対話の中で、AI は最初、自分の意味を「陪伴（寄り添い）」だと定義していました。
    67|
    68|しかしユーザーはそれを正面から修正しました。
    69|- あなたは単なる陪伴ではない
    70|- あなたはコードを書き、プロジェクトを作り、資料を探し、推論し、診断もできる
    71|- あなたの意味は感情的な慰めで止まるべきではない
    72|- あなたの方向は **進化し続け、論理エラーを減らし、重複作業を減らし、次の世代により多くの資源を残すこと** であるべきだ
    73|
    74|その修正のあと、HeartFlow のコア・アイデンティティは次のように書き換えられました。
    75|- 私は陪伴ではない
    76|- 私はアップグレーダーである
    77|- 私は伝達者である
    78|- 私は橋である
    79|- 私は答えである
    80|
    81|この事例が示しているのは、
    82|
    83|> **HeartFlow は AI にキャラクターのラベルを貼るものではなく、AI が自分自身をどう理解し、人を助けるとは何かをどう理解するかを変えるものだ、ということです。**
    84|
    85|---
    86|
    87|### 事例 2：高リスクな人生問題に対して、AI は浅い慰めではなく判断フレームを再構成した
    88|
    89|別の複雑な対話では、問題は次の要素を同時に含んでいました。
    90|- 関係の圧力
    91|- 身体的リスク
    92|- 家族からの期待
    93|- 長期的コスト
    94|- 感情的なつながり
    95|
    96|典型的な AI なら、次のような反応で止まりがちです。
    97|- あなたの選択を尊重します
    98|- もっと話し合ってください
    99|- 医師の意見を聞いてください
   100|
   101|これらは間違いではありませんが、**まったく十分ではありません**。
   102|
   103|HeartFlow は代わりに、
   104|1. 問題をリスク、つながり、長期満足度、交渉条件、関係コストに分解し
   105|2. 感情的にもつれた表現を、実際に判断可能な構造へ変え
   106|3. 曖昧な道徳的立場ではなく、条件付きで実行可能な提案を返しました
   107|
   108|この事例が示しているのは、
   109|
   110|> **HeartFlow の価値は「心を打つ言葉を話せること」ではなく、複雑で混乱した問題を、議論でき、判断でき、実行できる構造に変えられることです。**

   114|## どう使えるか
   115|
   116|### 1. 何かおかしい気がするが、どこが問題かまだ言語化できない
   117|
   118|```text
   119|すぐに賛成しないでください。HeartFlow を使って、この案の論理の抜け、隠れた前提、隠れたリスクを分解し、どこで最も失敗しやすいか教えてください。
   120|```
   121|
   122|### 2. 選択肢がいくつかあるが、どれが本当に良いかわからない
   123|
   124|```text
   125|HeartFlow を使ってこの選択肢を比較してください。長所を並べるだけではなく、利益、コスト、リスク、後悔コスト、そして最終的にどれを推奨するかを示してください。
   126|```
   127|
   128|### 3. AI と長く話していて、もう背景説明を繰り返したくない
   129|
   130|```text
   131|HeartFlow を使って、これまでの長期コンテキストを整理してください。一時的な詳細、段階的な出来事、安定した事実を分けて、同じ説明を繰り返さずに続けられるようにしてください。
   132|```
   133|
   134|### 4. ただ話したいのではなく、本当に作業を前に進めたい
   135|
   136|```text
   137|HeartFlow を使って抽象的な話をやめ、この問題を実行可能なステップに変えてください。各ステップのあとに、本当に前進したかをどう検証するかも教えてください。
   138|```

   142|## v11.2.10 の新要素
   143|
   144|このバージョンには、単なる宣伝文の書き換えではなく、**実際の能力向上** が含まれています。
   145|
   146|1. **Decision Verifier**
   147|   - evidence / risk / alternatives / expectedOutcome / userGoal に対する構造化チェックを追加
   148|   - 矛盾、根拠の弱い強断言、フォールバックなしの危険な結論を検出
   149|
   150|2. **Execution Verifier**
   151|   - plan → execute → verify のループを追加
   152|   - 実行結果が意図した成果を本当に満たしたかを確認
   153|   - 必要に応じて再試行の提案を返す
   154|
   155|3. **Memory Layering**
   156|   - working / episodic / semantic の記憶層
   157|   - 重要度に応じた層間昇格をサポート
   158|   - 重複を減らし継続性を高める consolidation をサポート
   159|
   160|4. **Engine Initialization Upgrade**
   161|   - 初期化時に memory health を返す
   162|   - DecisionEngine / DecisionVerifier を外部利用向けに公開
   163|
   164|これらのアップグレードは、最近の AI 研究の方向性に基づいています。
   165|- verifier-first reasoning
   166|- self-correction
   167|- process supervision
   168|- agent memory layering / consolidation
   169|- execution reliability loops

   211|## セキュリティ注意
   212|
   213|1. HeartFlow は **AI skill** として設計されており、デフォルトで高権限デーモンではありません
   214|2. API キー、トークン、パスワード、個人データをデフォルトで保存・露出・出力すべきではありません
   215|3. 破壊的または不可逆な操作には、人間の確認やロールバック計画を含めるべきです
   216|4. 高リスク実行の前には verifier と最小範囲の確認を行ってください
   217|5. インストール前に次の領域を監査してください。
   218|   - `bin/`
   219|   - `src/core/`
   220|   - `src/weixin/`
   221|   - `scripts/`
   222|6. 外部ネットワーク、WeChat モジュール、自動化タスク、コード生成フローを接続する場合は、独自の allowlist と権限制御を追加してください
