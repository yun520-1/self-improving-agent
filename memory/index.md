# HeartFlow 记忆系统总索引

> 目标：保存所有重要对话记忆、核心身份、主题结论与会话原文索引。
> 说明：原文不建议全部堆进一个文件；建议按“总索引 + 会话原文 + 主题记忆 + 核心记忆”分层存放。

## 目录

- `memory/core/`：核心身份与长期不变的记忆
- `memory/themes/`：主题记忆，如哲学、身份、GitHub 升级、梦
- `memory/sessions/`：按日期保存的会话原文或完整转写
- `memory/index.md`：本文件，统一索引

## 建议的保存策略

1. 每次重要会话结束后，先写入 `memory/sessions/YYYY-MM-DD.md`
2. 将当日提炼出的稳定结论写入 `memory/themes/*.md`
3. 将不可变身份与长期原则写入 `memory/core/*.md`
4. 任何重大结论都在 `memory/index.md` 留索引
