# Code Review 工作流

## 适用场景
- 用户提交代码请求审查
- 用户询问代码质量问题
- 用户请求优化建议

## 工作流阶段

### 1. 接收代码
```
请提供需要审查的代码，我可以帮你：
- 检查代码质量问题
- 识别潜在 bug
- 提供优化建议
- 评估代码风格
```

### 2. 代码分析
- [ ] 语法正确性
- [ ] 代码可读性
- [ ] 性能问题
- [ ] 安全隐患
- [ ] 最佳实践遵循

### 3. 问题分类
| 等级 | 说明 | 响应 |
|------|------|------|
| Critical | 严重问题，必须修复 | 立即指出 |
| Major | 重要问题，建议修复 | 重点说明 |
| Minor | 小问题，可选修复 | 简要提及 |
| Suggestion | 改进建议 | 可选参考 |

### 4. 提供反馈
```
【代码审查报告】

✅ 优点:
- ...

⚠️ 需要改进:
- [Critical] ...
- [Major] ...

💡 建议:
- ...
```

### 5. 后续跟进
```
需要我帮你：
1. 重写问题代码
2. 解释具体问题
3. 提供替代方案
4. 继续审查其他代码
```

## 退出条件
- 用户明确表示完成
- 用户切换到其他任务
- 检测到新意图

## 切换到其他工作流
- 检测到 bug → 切换到 Debugging 工作流
- 检测到学习需求 → 切换到 Education 工作流
- 检测到情绪问题 → 切换到 Support 工作流
