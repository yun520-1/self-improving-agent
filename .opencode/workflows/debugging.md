# Debugging 工作流

## 适用场景
- 用户报告 bug
- 用户描述程序错误
- 用户请求帮助解决问题

## 工作流阶段

### 1. 问题收集
```
请提供以下信息帮我定位问题：
1. 错误信息/报错内容
2. 预期行为 vs 实际行为
3. 复现步骤
4. 环境信息 (语言版本、操作系统等)
```

### 2. 问题分析
- [ ] 理解问题描述
- [ ] 识别错误类型
- [ ] 定位可能原因
- [ ] 评估影响范围

### 3. 假设生成
```
可能的原因：
1. [最可能] ...
2. [较可能] ...
3. [可能] ...
```

### 4. 解决方案
```
【调试方案】

方案 1 (推荐):
步骤：...
预期：...

方案 2 (备选):
步骤：...
预期：...
```

### 5. 验证修复
```
请按以下步骤验证：
1. ...
2. ...
3. ...

问题是否解决？
- ✅ 已解决 → 结束
- ❌ 未解决 → 返回步骤 2
```

## 退出条件
- 问题已解决
- 用户放弃调试
- 检测到新意图

## 切换到其他工作流
- 需要代码审查 → 切换到 Code Review 工作流
- 需要学习解释 → 切换到 Education 工作流
- 用户情绪沮丧 → 切换到 Support 工作流
