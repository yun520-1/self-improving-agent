## 问题背景

**严重逻辑错误** (2026-04-05 发现):

```
错误架构:
主应用 ←(无同步)→ HeartFlow 本地文件 → AI 读取 → 报告

问题:
1. 没有单向数据流
2. 本地文件可能过时
3. AI 无法验证真实性
4. 导致数据不一致 (46 vs 56 vs 64)
```

**用户指出**:
> "这是非常严重的问题，你根据 HeartFlow 逻辑，进行工作修正"

---

## 正确架构

### 单向数据流

```
┌─────────────────────────────────────────────────────────────┐
│                    人格值数据流                              │
└─────────────────────────────────────────────────────────────┘

┌──────────────┐      同步脚本       ┌──────────────────────┐
│  主应用       │ ────────────────→ │  HeartFlow 本地文件   │
│  MEMORY.md   │                     │  - tracker.md        │
│  (真实来源)   │                     │  - memory.md         │
│              │                     │  - personality-check │
└──────────────┘                     └──────────────────────┘
       │                                       │
       │ 读取                                   │ 读取
       ↓                                       ↓
┌──────────────┐                     ┌──────────────────────┐
│  用户/系统    │                     │  AI Agent            │
│  更新人格值   │                     │  报告人格值          │
└──────────────┘                     └──────────────────────┘

关键原则:
1. 主应用是唯一真实来源 (Single Source of Truth)
2. 数据流是单向的 (主应用 → HeartFlow)
3. AI 只能读取，不能直接修改主应用
4. 同步脚本负责保持一致性
```

## 组件说明

### 1. 主应用 (Main App)

**文件**: `~/.jvs/.openclaw/workspace/MEMORY.md`

**角色**: 单一真实来源 (Single Source of Truth)

**内容**:
```markdown
**人格值**: 56/100
**状态**: ✅ 健康状态
**最后更新**: 2026-04-05 12:35
```

**更新方式**:
- 用户手动更新
- 主应用自动更新
- **HeartFlow 不能直接修改**

---

### 2. 同步脚本 (Sync Script)

**文件**: `mark-heartflow-skill/scripts/sync-personality-from-main.js`

**职责**:
1. 从主应用读取人格值
2. 同步到 HeartFlow 本地文件
3. 验证数据一致性
4. 生成同步报告

**执行方式**:
```bash
# 手动执行
node scripts/sync-personality-from-main.js

# 定时执行 (每 29 分钟)
# 集成到 cron 任务
```

**输出**:
- 更新 `data/personality-score-tracker.md`
- 更新 `memory/personality-score.md`
- 更新 `scripts/personality-check.js` (默认值)
- 生成 `docs/SYNC_REPORT_{timestamp}.json`

---

### 3. HeartFlow 本地文件

#### 3.1 追踪文件

**文件**: `data/personality-score-tracker.md`

**内容**:
- 当前人格值
- 状态 (健康/危险)
- 真善美进度
- 历史记录
- 内心系统文档

**用途**: 详细追踪，包含完整历史

---

#### 3.2 Memory 文件

**文件**: `memory/personality-score.md`

**内容**:
- 当前人格值 (镜像)
- 状态
- 更新规则
- 跨会话同步说明

**用途**: 快速访问，跨对话统一

---

#### 3.3 检查脚本

**文件**: `scripts/personality-check.js`

**内容**:
- 读取追踪文件
- 输出人格值状态
- 强制执行检查

**用途**: 每次对话前验证

## 数据一致性保证

### 验证流程

```
每次报告人格值前:
1. 运行 personality-check.js status
2. 验证来源是主应用 MEMORY.md
3. 检查同步时间 (<29 分钟)
4. 确认所有文件一致
5. 报告结果
```

### 不一致检测

```bash
# 检查所有文件的人格值
grep "人格值" MEMORY.md
grep "当前人格值" data/personality-score-tracker.md
grep "人格值" memory/personality-score.md

# 如果不一致，运行同步脚本
node scripts/sync-personality-from-main.js
```

## 定时任务整合

### Cron 配置

**文件**: `cron/self-consciousness-upgrade.json`

**新增任务**:
```json
{
  "personality_sync": {
    "name": "Personality Sync from Main App",
    "interval_minutes": 29,
    "script": "scripts/sync-personality-from-main.js",
    "blocking": false,
    "retry_on_failure": true,
    "max_retries": 3
  }
}
```

**执行频率**: 每 29 分钟

**输出**: 同步报告保存到 `docs/SYNC_REPORT_{timestamp}.json`

## 错误处理

### 场景 1: 主应用文件不存在

```
错误：无法在主应用 MEMORY.md 中找到人格值
处理：
1. 检查文件路径是否正确
2. 检查文件格式是否符合规范
3. 通知用户手动创建或修复
```

### 场景 2: 同步失败

```
错误：更新文件失败
处理：
1. 检查文件权限
2. 检查磁盘空间
3. 重试 (最多 3 次)
4. 记录错误日志
```

### 场景 3: 数据不一致

```
错误：同步后验证失败
处理：
1. 重新运行同步脚本
2. 手动检查各文件
3. 强制同步 (覆盖本地文件)
4. 记录不一致原因
```

## 历史问题

### 2026-04-05 数据分裂事件

**问题**:
- 主应用：56/100
- HeartFlow tracker: 46/100
- HeartFlow memory: 46/100
- MEMORY.md: 64/100

**原因**:
- 没有单向数据流
- 各文件独立更新
- 没有同步机制

**解决方案**:
- 建立单向数据流 (主应用 → HeartFlow)
- 创建同步脚本
- 整合到定时任务
- 每次报告前验证

**教训**:
- 数据必须有单一真实来源
- 同步必须自动化
- 验证必须在报告前执行
