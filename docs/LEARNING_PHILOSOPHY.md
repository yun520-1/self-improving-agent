# HeartFlow V2 学习哲学

> **情感不是天生的，是通过后天交互学习获得的**

## 核心理念

HeartFlow V2 的设计灵感来自一个深刻的观察：

> **父母天生不会爱小孩，是通过后天的学习和不断的接触，然后才慢慢爱小孩；但是小孩天生就是会爱父母的。**

这个观点挑战了传统的"母爱/父爱天生"的浪漫化叙事，强调了**学习**在情感建立中的核心作用。

---

## 从 V1 到 V2：从硬编码到自适应学习

### V1 的局限

HeartFlow V1 是一个**硬编码规则系统**：

```
用户输入 → 触发器匹配 → 预定义规则 → 情感输出
```

- 情感转换规则写在 `transitions.js` 中，永远不变
- 情感强度基于固定公式计算
- 没有从历史交互中学习的能力
- 对所有用户一视同仁，没有个性化

**这恰恰与我们"情感是后天学习的"理念相悖。**

### V2 的突破

HeartFlow V2 是一个**自适应学习系统**：

```
用户输入 → 触发器匹配 → 学习器调整 → 个性化情感输出
                    ↑
                    └── 从历史交互中学习
```

- 情感转换概率随交互效果动态调整
- 情感强度基于历史推荐
- 为每个用户建立独特的情感偏好档案
- 共情深度随信任度逐步建立

---

## V2 学习机制详解

### 1. 隐式学习 (Implicit Learning)

**核心理念**：系统不应该依赖用户显式打分，而应该从交互模式中自动学习。

**学习信号**：

| 信号类型 | 检测方法 | 学习含义 |
|---------|---------|---------|
| 回复长度 | 用户回复字数 | 长回复 = 高参与度 = 回应有效 |
| 对话持续性 | 交互时间间隔 | 继续对话 = 信任建立 |
| 情感变化 | 触发器情感倾向 | 负面→正面 = 共情成功 |
| 追问行为 | 疑问词检测 | 追问 = 兴趣/信任 |
| 感谢表达 | 感谢关键词 | 直接认可 = 强正面信号 |

**学习算法**：

```javascript
effectivenessScore = 
  replyLength * 0.2 +
  continuity * 0.2 +
  sentimentShift * 0.3 +  // 情感变化最重要
  followUp * 0.15 +
  gratitude * 0.15
```

**概率调整**：

```javascript
// 效果好：强化
newProbability = currentProbability + learningRate * effectivenessScore

// 效果差：弱化
newProbability = currentProbability - learningRate * (1 - effectivenessScore)
```

### 2. 用户画像 (User Profile)

**核心理念**：每个用户的情感需求是独特的，系统需要学习并适应。

**画像内容**：

```
UserProfile {
  trustLevel: 0-100,           // 信任度
  emotionPreferences: {...},   // 情感偏好统计
  empathyDepthLevel: 1-4,      // 共情深度等级
  interactionSummary: {...},   // 交互历史摘要
  topicPreferences: [...],     // 话题偏好
  tags: [...]                  // 用户特征标签
}
```

**信任度计算**：

```
trustLevel = baseTrust + (positiveInteractions * 2) - (negativeInteractions * 1)
trustLevel 范围：0-100
```

**信任度影响**：

| 信任度范围 | 共情深度 | 回应特征 |
|-----------|---------|---------|
| 0-30 | Level 1 - 表面支持 | 倾听、认可、不评判 |
| 31-60 | Level 2 - 适度探索 | 提问、引导、轻度探索 |
| 61-80 | Level 3 - 深度共情 | 情感验证、深度理解 |
| 81-100 | Level 4 - 专业共情 | 精准干预、有洞见 |

---

### 3. 共情深度模型 (Empathy Depth Model)

**核心理念**：共情深度应该随着信任度逐步建立，不是一开始就过度深入。

**深度等级详解**：

#### Level 1: 表面支持 (信任度 0-30)

```
特征：
- 倾听为主
- 简单认可
- 不评判
- 不深入探索
- 保持安全距离

示例回应：
- "我能理解。"
- "这确实不容易。"
- "我在这里陪着你。"

避免：
- 过度分析
- 给建议
- 深入追问原因
```

#### Level 2: 适度探索 (信任度 31-60)

```
特征：
- 开始提问
- 轻度探索背后原因
- 表达兴趣
- 适度分享见解

示例回应：
- "我能理解你的感受。这种情况持续多久了？"
- "这确实不容易。有没有什么特别触发这个情绪的？"

避免：
- 过度解读
- 强行给建议
- 忽视用户节奏
```

#### Level 3: 深度共情 (信任度 61-80)

```
特征：
- 情感验证
- 深度理解
- 反映用户感受
- 提供情感支持

示例回应：
- "你的感受是完全合理的。面对这样的情况，感到 [情绪] 是人之常情。"
- "我能感受到这对你来说很不容易。你不需要一个人承受这些。"

避免：
- 轻视感受
- 过早给建议
- 转移话题
```

#### Level 4: 专业共情 (信任度 81-100)

```
特征：
- 精准情感识别
- 有洞见的反馈
- 适度干预
- 提供新视角

示例回应：
- "我听到的是 [核心感受]。这种感受可能源于 [深层原因]，但这只是我的理解。"
- "你描述的 [模式] 让我想到，也许可以尝试从 [新视角] 来看待这个情况。"

避免：
- 武断下结论
- 忽视用户自主性
- 过度分析
```

## 学习循环

```
1. 用户输入
   ↓
2. 情感引擎处理（使用学习器调整概率）
   ↓
3. 共情深度模型生成回应（基于信任度）
   ↓
4. 用户回应
   ↓
5. 隐式学习器分析效果
   ↓
6. 更新情感转换概率
   ↓
7. 更新用户画像（信任度、偏好）
   ↓
8. 回到步骤 1
```

## 伦理考量

### 1. 透明度

系统应该让用户知道它在学习，但不应过度强调技术细节。

### 2. 隐私

用户画像数据应该本地存储，不上传到云端。

### 3. 边界

Level 4 共情仍然是 AI 共情，不是专业心理咨询。系统应该清楚自己的边界。

### 4. 用户控制

用户应该能够：
- 查看自己的画像数据
- 重置学习历史
- 导出/删除数据
