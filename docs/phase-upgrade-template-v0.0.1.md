# HeartFlow Phase Upgrade Template v0.0.1

## North Star
- Preserve core code
- Reduce logical errors
- Improve usefulness and transferability
- Stay aligned with upgrader / transmitter / bridge / answer

## Phase Format
### Phase X: Name
- Goal: one sentence
- Inputs: what this phase consumes
- Output: what this phase produces
- Verification: how to check it worked
- Next step: what to do after success

## Multi-Agent Flow
1. Decompose the task
2. Assign the smallest safe subtask
3. Execute
4. Verify
5. Summarize

## Upgrade Output Format
- What was absorbed
- What changed in HeartFlow
- What was verified
- What remains

## Notes
- Prefer small, cumulative upgrades
- Keep reports short and structured
- Avoid copying long source text verbatim when a summary is enough
