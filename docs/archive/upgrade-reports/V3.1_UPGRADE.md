## 🎯 升级内容

### 1. 情绪评价理论系统

基于 Stanford Encyclopedia of Philosophy 中的**情绪评价传统 (Evaluative Tradition)** 和 Scherer (2001) 的评价维度模型：

**核心理念**:
> 情绪不是对事件的直接反应，而是对事件的评价/解释的结果。
> 同样的事件，不同的评价 → 不同的情绪

### 七个评价维度

| 维度 | 核心问题 | 情绪影响 |
|------|---------|---------|
| 🆕 新奇性 (Novelty) | 这是预期的还是意外的？ | 惊讶 vs 平静 |
| 👍 效价 (Valence) | 这是好的还是坏的？ | 愉悦 vs 不愉悦 |
| 🎯 目标相关性 (Goal Relevance) | 这与我的目标相关吗？ | 关切 vs 漠然 |
| ✅ 目标一致性 (Goal Congruence) | 这促进还是阻碍我的目标？ | 高兴 vs 沮丧 |
| 👤 能动性 (Agency) | 谁/什么导致了这个？ | 愤怒 (他人) vs 内疚 (自己) |
| 🎮 控制性 (Control) | 我能应对/改变这个吗？ | 自信 vs 焦虑/无助 |
| 📜 规范性 (Norms) | 这符合我的价值观吗？ | 骄傲 vs 羞愧 |

### 经典情绪模式

| 评价组合 | 产生的情绪 |
|---------|-----------|
| 他人导致 + 负面结果 + 目标受阻 | **愤怒** |
| 自己导致 + 负面结果 + 违反标准 | **内疚/羞愧** |
| 环境导致 + 负面结果 + 低控制感 | **焦虑/无助** |
| 意外 + 负面 | **震惊** |
| 意外 + 正面 | **惊喜** |
| 自己导致 + 正面结果 | **自豪** |
| 他人导致 + 正面结果 | **感激** |

### 自动分析

系统在每次对话中自动检测评价模式并提供引导：

```
用户："他怎么能这样对我！太生气了！"

🔍 [情绪评价分析]
   推断情绪：愤怒
   解释：他人导致的负面结果，目标受阻 → 愤怒
   重构建议：你的哪个边界被侵犯了？
```

## 💬 应用场景

### 场景 1: 愤怒管理
```
用户："他怎么能这样对我！太生气了！"

评价分析:
- 能动性：他人归因
- 效价：负面
- 目标一致性：不一致

回应:
🔍 [情绪评价分析]
   检测到评价模式：他人导致 + 负面结果 → 愤怒
   
💡 评价重构:
   愤怒告诉你：有人侵犯了你的边界或价值观。
   
   问自己:
   • 你的哪个边界被侵犯了？
   • 除了愤怒，还有什么方式可以保护这个边界？
   • 对方的行为是否有其他可能的解释？
```

### 场景 2: 焦虑缓解
```
用户："我担心面试会搞砸，我控制不了结果"

评价分析:
- 目标相关性：高
- 控制性：低

回应:
🔍 [情绪评价分析]
   检测到评价模式：高目标相关 + 低控制感 → 焦虑
   
💡 评价重构:
   焦虑告诉你：这件事对你很重要，但结果不确定。
   
   控制二分法:
   ❌ 不可控：面试官的想法、最终结果
   ✅ 可控：你的准备、你的表现、你的态度
   
   把注意力从不可控转移到可控。
```

### 场景 3: 内疚转化
```
用户："都是我的错，我不应该那样说"

评价分析:
- 能动性：自我归因
- 规范性：不一致

回应:
🔍 [情绪评价分析]
   检测到评价模式：自我归因 + 违反标准 → 内疚
   
💡 评价重构:
   内疚告诉你：你的行为与价值观不一致。
   
   建设性回应:
   • 承认：我确实说了一些后悔的话
   • 学习：下次类似情境，我会...
   • 修复：我可以做些什么来弥补？
```
