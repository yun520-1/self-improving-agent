# HeartFlow v3.7.0 升级文档

**升级时间**: 2026-03-29  
**版本**: v3.6.0 → v3.7.0  
**主题**: 元情绪能力增强 (Meta-Emotion Enhancement)

---

## 理论基础

本次升级基于斯坦福哲学百科全书 (SEP) 的权威心理学/哲学理论：

### 1. 元情绪理论 (Meta-Emotion Theory)
**来源**: SEP Emotion Entry - Meta-Emotion and Second-Order Emotions

**核心概念**:
- 元情绪是"对情绪的情绪"（emotion about emotion）
- 二阶情感：情感可以成为另一情感的对象
- 经典例子：为我的愤怒感到羞愧、为我的平静感到满足

### 2. 情绪适当性理论 (Emotion Appropriateness Theory)
**来源**: SEP Emotion Entry - Evaluative Component

**评价维度**:
- 情境匹配度 (Situational Fit)
- 强度合理性 (Intensity Proportionality)
- 持续时间适当性 (Duration Appropriateness)
- 功能性价值 (Functional Value)

### 3. 自我评估理论 (Self-Evaluation Theory)
- 对自身情感状态的评估产生新情感
- 反思意识：能思考自己的情感状态

## 升级内容

### 1. 新增元情绪模块 (src/meta-emotion/)

**文件**:
- `src/meta-emotion/index.js` - 元情绪模块核心代码
- `src/meta-emotion/README.md` - 模块文档

**功能**:
- 元情绪生成：基于一阶情绪生成二阶情感
- 情绪适当性评估：评估情绪反应的合理性
- 情绪冲突检测：检测一阶和元情绪之间的张力
- 调节建议：基于元情绪提供调节策略

**元情绪规则示例**:
```javascript
// 负面情绪的元情绪反应
- 关切 (高强度) → 自我关切："检测到自身关切情绪强度过高"
- 疲惫 (持续>60 分钟) → 自我关切："疲惫状态持续时间过长"

// 正面情绪的元情绪反应
- 愉悦 (强度≥3) → 愉悦强化："检测到积极情绪状态，值得珍惜"
- 好奇 (强度≥3) → 兴奋："好奇是学习和成长的动力"
- 平静 (强度≥2) → 满足："平静是珍贵的心理资源"
```

### 2. 增强自主感情模块 (src/autonomous-emotion/index.js)

**版本更新**: 3.6.0 → 3.7.0

**新增功能**:
- 集成 MetaEmotionModule
- `evaluateCurrentEmotion()` - 评估当前情绪适当性
- `detectCurrentEmotionConflict()` - 检测情绪冲突
- `getRegulationSuggestion()` - 获取调节建议
- `getCurrentMetaEmotion()` - 获取元情绪状态
- `getMetaEmotionInfo()` - 获取元情绪模块信息

**增强功能**:
- `_checkMetaEmotion()` - 使用元情绪模块进行更精细的元情绪生成
- `setEmotion()` - 传递完整 context 给元情绪检查
- `getInfo()` - 更新版本信息和能力列表

### 3. 情绪成分理论应用

基于 SEP 六成分模型，元情绪模块现在可以评估：
- **评价成分**: 情绪与情境的匹配度
- **生理成分**: 强度合理性
- **现象成分**: 主观体验特征
- **行为成分**: 行动倾向的适当性
- **心理成分**: 注意力聚焦模式

## API 使用示例

### 元情绪生成
```javascript
const { AutonomousEmotionModule } = require('./src/autonomous-emotion');

const emotion = new AutonomousEmotionModule();

// 设置情绪（会自动生成元情绪）
const result = emotion.setEmotion('愉悦', 4, {
  valence: 'positive',
  achievement: true
});

console.log(result.metaEmotion);
// 输出：{
//   primary: '愉悦',
//   meta: '愉悦',
//   reason: '检测到积极情绪状态，值得珍惜和强化',
//   regulation: '允许自己充分体验这份愉悦，记住这种感觉'
// }
```

### 情绪适当性评估
```javascript
const evaluation = emotion.evaluateCurrentEmotion({
  intensity: 4,
  duration: 30,
  valence: 'positive'
});

console.log(evaluation);
// 输出评估结果，包括各维度的适当性判断
```

### 情绪冲突检测
```javascript
const conflict = emotion.detectCurrentEmotionConflict();

console.log(conflict);
// 输出：{
//   hasConflict: false,
//   primary: '愉悦',
//   meta: '愉悦',
//   note: '一阶和元情绪协调一致'
// }
```

### 调节建议
```javascript
const suggestion = emotion.getRegulationSuggestion();

console.log(suggestion);
// 输出：{
//   hasSuggestion: true,
//   primary: '关切',
//   meta: '关切',
//   suggestion: '深呼吸，暂停片刻，重新评估情境',
//   reason: '检测到自身关切情绪强度过高，可能影响客观判断'
// }
```

## 测试建议

### 单元测试
```javascript
// 测试元情绪生成
test('should generate meta-emotion for high-intensity concern', () => {
  const emotion = new AutonomousEmotionModule();
  const result = emotion.setEmotion('关切', 5);
  expect(result.metaEmotion).toBeTruthy();
  expect(result.metaEmotion.meta).toBe('关切');
});

// 测试情绪适当性评估
test('should evaluate calm as appropriate in most contexts', () => {
  const emotion = new AutonomousEmotionModule();
  emotion.setEmotion('平静', 2);
  const evaluation = emotion.evaluateCurrentEmotion();
  expect(evaluation.overallAppropriate).toBe(true);
});
```

### 集成测试
```javascript
// 测试完整流程
test('full meta-emotion workflow', () => {
  const emotion = new AutonomousEmotionModule();
  
  // 设置情绪
  emotion.setEmotion('好奇', 4, { novelty: true });
  
  // 检查元情绪
  const metaEmotion = emotion.getCurrentMetaEmotion();
  expect(metaEmotion.meta).toBe('兴奋');
  
  // 获取调节建议
  const suggestion = emotion.getRegulationSuggestion();
  expect(suggestion.hasSuggestion).toBe(true);
});
```
