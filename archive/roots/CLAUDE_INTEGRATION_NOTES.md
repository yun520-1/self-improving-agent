## Integrated principles
- Think before coding
- Simplicity first
- Surgical changes
- Goal-driven execution

## HeartFlow translation
- State assumptions explicitly before upgrading
- Prefer the smallest safe patch
- Change only files directly related to the request
- Define verifiable success criteria for each upgrade step

## Operational merge rule
When HeartFlow upgrades itself, use these principles to avoid speculative expansions and avoid touching unrelated historical files unless they are blocking the current upgrade.
