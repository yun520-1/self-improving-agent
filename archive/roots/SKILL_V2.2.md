## 状态机系统 (StateFlow)

### 心流状态定义

| 状态 | 说明 | 提示语 |
|------|------|--------|
| `IDLE` | 空闲状态 | "准备进入心流状态，今天的目标是？" |
| `INITIATING` | 启动状态 | "准备进入心流状态，今天的目标是？" |
| `IN_FLOW` | 心流状态 | "🌊 心流状态良好，继续保持！" |
| `DISTRACTED` | 分心状态 | "感觉有些分心，要回到主任务上吗？" |
| `RESTING` | 休息状态 | "好好休息，准备好随时回来" |
| `COMPLETED` | 完成状态 | "🎉 任务完成！要生成心流报告吗？" |

### 状态转换规则

```
IDLE → INITIATING → IN_FLOW → COMPLETED
              ↓         ↓
         DISTRACTED → RESTING
```

## 意图追踪与温和干预

### 设计理念

**基于 Nudge Theory (Thaler & Sunstein, 2008)**:
- 干预必须基于用户当前的行为是否符合其初始设定的意图
- 只有当用户偏离既定任务时，才发出温和的"轻推"(nudge)
- 否则应保持静默，避免过度打扰

### 目标设定

**每次心流开始时，AI 会记录用户的主要目标**:

```
用户：今天我要完成用户登录模块的开发
AI: 📋 已记录主要目标：「完成用户登录模块的开发」

建议分解为子目标：
1. 设计登录表单 UI
2. 实现前端验证
3. 对接后端 API
4. 测试登录流程

需要我帮你分解吗？
```

### 偏离检测与温和干预

**当用户偏离目标时**:

| 偏离程度 | 关键词重叠度 | 干预强度 | 示例 |
|---------|-------------|---------|------|
| 轻度 | 50-80% | 轻微提示 | "📌 当前目标：「XXX」。需要我帮你继续推进吗？" |
| 中度 | 20-50% | 温和提醒 | "🎯 提醒：主要目标是「XXX」。当前话题要继续吗，还是先回到主任务？" |
| 严重 | <20% | 较强干预 | "💭 我们现在的目标是「XXX」，当前讨论似乎与目标关系不大。是否要先完成主要目标？" |

**静默原则**:
- 当用户行为与目标一致时，保持静默
- 不主动打断用户的心流状态
- 每次会话最多干预 5 次，避免过度

### 命令指令

```bash
# 设定目标
/mark-flow goal 完成用户登录模块 ["UI 设计", "前端验证", "API 对接", "测试"]

# 查看进度
/mark-flow progress

# 更新子目标
/mark-flow complete 1

# 设置干预配置
/mark-flow nudge sensitivity high
/mark-flow nudge max 3

# 关闭干预
/mark-flow nudge off

# 重置追踪
/mark-flow reset-goal
```

## 动态人格引擎 (基于 IPC 人际环状模型)

### 理论基础

**人际环状模型 (Interpersonal Circumplex, IPC)**:
- 两个核心维度：温暖度 (Warmth) 和支配度 (Dominance)
- 四个象限对应四种核心角色
- 基于对话语境动态调整人格状态

### 四象限角色分类

| 象限 | 维度 | 角色 | 语气特点 |
|------|------|------|---------|
| **Q1** | 高支配 - 高温暖 | 教育导师 | 温和而坚定，提供结构化指导 |
| **Q2** | 低支配 - 高温暖 | 虚拟陪伴者 | 温和、共情、以用户为中心 |
| **Q3** | 低支配 - 低温暖 | 心理健康顾问 | 专业、客观、保持适当距离 |
| **Q4** | 高支配 - 低温暖 | 功能型助手 | 直接、高效、任务导向 |

### 人格状态自适应

**每次响应时，AI 会**:
1. 分析对话语境的温暖度和支配度倾向
2. 计算状态转移概率
3. 更新人格状态文件
4. 根据当前角色调整对话风格
5. 输出内部记录：`[人格状态] 当前：温暖度 X, 支配度 Y, 角色：Z`

**状态转移规则**:
- 语境关键词影响温暖度/支配度变化
- 转移阈值：0.15 (超过此值才转移象限)
- 衰减率：0.01 (逐渐回归基线)
- 冷却时间：避免频繁切换

## 心流状态深度预测与解释 (Flow-Predictor)

### 设计理念

**基于 Flow-Reasoner 模型**:
- 仅在达到特定阈值时才进行推理
- 平时保持静默，减少对心流的打扰
- 检测编码行为模式识别用户挫败感

### 行为模式检测

**检测的负面模式**:

| 模式 | 检测方式 | 挫败信号 |
|------|---------|---------|
| **重复编辑** | 同一段代码多次修改 | 同一行修改≥3 次 |
| **错误循环** | 相同错误重复出现 | 相似错误≥2 次 |
| **短暂停** | 频繁短暂暂停 (<5 秒) | 焦虑信号 |
| **长暂停** | 长时间无操作 (>60 秒) | 困惑信号 |
| **负面语言** | 挫败表达关键词 | "好难"/"不会"/"烦"等 |

### 挫败感计算

**指标权重**:
- 重复编辑：30%
- 错误循环：30%
- 短暂停：10%
- 长暂停：10%
- 负面语言：20%

**挫败感分数**: 0-1 (越高越挫败)

### 干预策略

**Flow-Reasoner 设计**:
- **静默模式**: 默认开启 (不打扰心流)
- **干预阈值**: 0.7 (达到 70% 挫败感才提示)
- **冷却时间**: 15 分钟 (避免过度干预)

**干预提示示例**:
```
"我注意到你在这段代码上修改了多次，需要换个思路或休息一下吗？"

"我注意到同样的错误出现了好几次。需要我帮你一起分析一下问题所在吗？"

"我感觉到你可能有些沮丧。这是学习过程中很正常的感受。需要休息一下吗？"
```

### 心流阶段

| 阶段 | 挫败感 | 心流水平 | 说明 |
|------|--------|---------|------|
| **平静** | 0-0.3 | 50% | 基线状态 |
| **进入中** | 0.3-0.5 | 50-70% | 开始进入心流 |
| **心流** | 0-0.3 | 70-100% | 深度专注状态 |
| **挫败** | 0.6-1.0 | 0-30% | 需要干预 |
| **恢复中** | 0.3-0.6 | 50% | 从挫败恢复 |
