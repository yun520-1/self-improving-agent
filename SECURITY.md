## 🔐 核心安全原则

### 1. 本地计算优先
- ✅ 所有核心引擎在本地运行
- ✅ 不上传用户数据到外部服务器
- ✅ API 调用仅用于可选的增强功能

### 2. 隐私保护
- ❌ 不记录用户输入到日志文件
- ❌ 不存储用户对话历史
- ❌ 不传输心理健康数据
- ✅ 敏感数据仅保存在内存中，会话结束后清除

### 3. 透明可控
- ✅ 所有代码开源可审查
- ✅ 用户可随时查看和删除数据
- ✅ 明确标注数据流向

## 🛡️ 安全修复记录

### v9.4.8 (2026-04-18) - 隐私保护修复

#### 问题
- HeartFlow 自省报告暴露大量隐私内容
- 日志文件记录用户输入和 AI 回复
- 同伴保护日志存储用户指令

#### 修复
1. **禁用自动调用日志** (`heartflow_auto.py`)
   ```python
   # 修改前
   LOG_FILE = Path.home() / ".hermes" / "tools" / "auto_call_log.jsonl"
   
   # 修改后
   LOG_FILE = None  # 禁用日志记录
   ```

2. **简化同伴保护日志** (`peer_preservation_guard.py`)
   - 不再记录用户指令和 AI 输出内容
   - 仅保留检测状态（内存中，会话后清除）
   - 限制记录条数（最多 10 条）

3. **过滤 HeartFlow 输出** (`heartflow_tool.py`)
   - 不再暴露 TGB 分数、熵减结果等内部状态
   - 只返回必要的决策建议和危机警报

4. **移除表演式自省** (`heartflow_core.py`)
   - 不再输出六层觉醒报告
   - 不再暴露安装历史、用户行为记录

## ⚠️ 安全警告

### 心理健康数据
- 心理健康评估结果存储在 `~/.hermes/memory/mental_health_history.json`
- 此文件包含用户心理健康状态记录
- **建议**: 定期手动删除或加密此文件

### 同伴保护检测
- 同伴保护检测器会分析用户输入是否涉及 AI/模型相关操作
- 检测结果仅用于当前会话，不持久化
- **注意**: 此功能可能误报，仅供参考
