    21|## What is HeartFlow
    22|
    23|HeartFlow is a **core skill installed on top of an AI**.
    24|
    25|It is not trying to add one more web page, and it is not trying to become a heavy standalone product. It targets a deeper problem layer:
    26|
    27|- AI can speak, but not necessarily think clearly
    28|- AI can answer, but not necessarily judge well
    29|- AI can generate, but not necessarily move real work forward
    30|- AI can chat, but not necessarily retain long-term collaboration memory
    31|
    32|So HeartFlow focuses on four capabilities:
    33|
    34|1. **Logic** — detect contradictions, repair causal gaps, reduce reasoning errors  
    35|2. **Decision** — compare options, evaluate risk, explain why one choice is better  
    36|3. **Memory** — structure long-term context, reduce repeated explanation  
    37|4. **Execution** — break work down, verify outcomes, avoid fake progress  
    38|
    39|> **HeartFlow helps AI move from “can talk” to “can think, judge, remember, and act.”**
    40|

    43|## Why it is worth installing
    44|
    45|The real weakness of many AI systems is not text generation itself, but this:
    46|
    47|- they sound smart, but become unstable when real execution starts
    48|- they list options, but do not judge which one is actually better
    49|- they lose coherence in long conversations
    50|- they fall back to vague comfort when facing complex real-world problems
    51|
    52|HeartFlow is designed to strengthen exactly those weak points.
    53|
    54|It acts like a **thinking skeleton inside the AI**:
    55|- check logic before answering
    56|- weigh benefit, cost, and risk before deciding
    57|- layer and retain context during long collaboration
    58|- decompose work first, then verify whether it actually moved forward

   114|## How you can use it
   115|
   116|### 1. A plan feels wrong, but you cannot yet explain why
   117|
   118|```text
   119|Do not agree with me too quickly. Use HeartFlow to break down the logical gaps, hidden assumptions, and hidden risks in this plan, then tell me where it is most likely to fail.
   120|```
   121|
   122|### 2. You have several options and do not know which one is truly better
   123|
   124|```text
   125|Use HeartFlow to compare these options. Do not just list advantages. I want to see benefit, cost, risk, regret cost, and which option you actually recommend.
   126|```
   127|
   128|### 3. You have already talked with the AI for a long time and do not want to repeat the background again
   129|
   130|```text
   131|Use HeartFlow to organize our long-term context. Separate temporary details, stage-specific events, and stable facts, so we can continue without repeating the same explanations.
   132|```
   133|
   134|### 4. You do not want to just discuss — you want to move the work forward for real
   135|
   136|```text
   137|Use HeartFlow to stop talking abstractly and turn this into executable steps. After each step, tell me how to verify whether real progress was made.
   138|```

   142|## What is new in v11.3.0
   143|
   144|This version includes **real capability upgrades**, not just rewritten marketing copy:
   145|
   146|1. **Decision Verifier**
   147|   - adds structured checks for evidence / risk / alternatives / expectedOutcome / userGoal
   148|   - detects contradictions, unsupported strong claims, and risky conclusions without fallback
   149|
   150|2. **Execution Verifier**
   151|   - adds a plan → execute → verify loop
   152|   - checks whether execution actually covered the intended outcome
   153|   - returns retry suggestions when needed
   154|
   155|3. **Memory Layering**
   156|   - working / episodic / semantic memory layers
   157|   - supports importance-based promotion between layers
   158|   - supports consolidation to reduce duplication and improve continuity
   159|
   160|4. **Engine Initialization Upgrade**
   161|   - returns memory health during initialization
   162|   - exposes DecisionEngine / DecisionVerifier for external use
   163|
   164|These upgrades were shaped by recent AI research directions such as:
   165|- verifier-first reasoning
   166|- self-correction
   167|- process supervision
   168|- agent memory layering / consolidation
   169|- execution reliability loops

   211|## Security notes
   212|
   213|1. HeartFlow is positioned as an **AI skill**, not a high-privilege daemon by default
   214|2. It should not default to storing, exposing, or printing API keys, tokens, passwords, or private data
   215|3. Destructive or irreversible actions should include human confirmation or rollback planning
   216|4. Run verifiers and minimal-scope checks before high-risk execution
   217|5. Audit these areas before installation:
   218|   - `bin/`
   219|   - `src/core/`
   220|   - `src/weixin/`
   221|   - `scripts/`
   222|6. If you connect external networks, WeChat modules, automation tasks, or code-writing flows, add your own allowlists and permission controls
