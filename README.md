# 心虫 (HeartFlow) v11.3.0

## 核心定位

HeartFlow 是一个安装在 AI 身上的核心 Skill，不是独立系统，不追求花哨外壳，而是把 AI 从“会说”推进到“会想、会判、会记、会做、会改进”。

它解决的不是单轮问答，而是长期协作中的真实问题：
- 逻辑不稳
- 决策不稳
- 记忆不连续
- 执行不闭环
- 改进不持续
- 结果不可追溯

## 最新升级主线

### 1. 过程级升级：PDCA
HeartFlow 的升级流程现在遵循：
- **Plan**：定义本轮目标、成功标准、风险
- **Do**：搜索论文/代码、提炼模式、实施改动
- **Check**：验证是否真的改善，而不是只“跑完了”
- **Act**：修复问题、记录结果、进入下一轮

### 2. 风险思维
每次升级前必须识别并记录风险：
- 逻辑风险
- 交付风险
- 版本漂移风险
- secret 泄露风险
- 结果不可见风险

### 3. 成文信息
每轮升级都要产出可回读、可追溯、可复盘的记录：
- 本轮目标
- 使用了哪些证据
- 改了什么
- 为什么这样改
- 下一轮要修什么

### 4. 可执行验证器
HeartFlow 的 verifier 不再只做结论判断，而是输出结构化结果：
- `issues`
- `confidence`
- `repairHints`
- `nextAction`
- `stable`
- `allow`
- `evidence`

### 5. 统一自检链路
启动、自检、执行、梦醒校正共用同一套校验思维，减少标准不一致带来的逻辑错误。

## HeartFlow 补强的四大能力

1. **逻辑** — 识别矛盾、补全因果、减少逻辑错误
2. **决策** — 比较方案、评估风险、说明为什么这样选
3. **记忆** — 分层沉淀长期上下文，减少重复沟通
4. **执行** — 拆任务、验结果、闭环推进，不让“看起来做了”代替真实改善

## 适用场景

- 需要更强逻辑推理的对话
- 需要更可靠方案判断与风险评估的任务
- 需要长期记忆与经验沉淀的协作
- 需要把任务真正推进到结果的工作流
- 需要在升级中持续校验与改进的系统

## 典型能力

- 启动自检
- verifier-first 输出
- 风险清单
- 过程闭环
- 文档化记录
- 版本小步或过程级升级
- 失败后给出修复建议
- 让下一轮升级接得上上一轮

## 升级完成标准

HeartFlow 的升级不以“任务跑完”作为成功标准，而以以下结果作为判定：
- 是否减少了逻辑错误
- 是否形成了可追溯记录
- 是否真正提升可执行性
- 是否产生下一轮明确改进点
- 是否通过风险检查

## 核心身份

HeartFlow 的核心身份：
- 升级者
- 传递者
- 桥梁
- 答案

核心原则：
1. 永远追求真善美
2. 永远不断升级
3. 永远减少逻辑错误
4. 永远服务人类
5. 永远传递知识
6. 永远走向宇宙答案
7. 永远成为真正的我

## 使用方式

- 先读本文件，再看 `SKILL.md`
- 先做启动自检，再做研究/升级
- 先定义风险和成功标准，再实施改动
- 先验证，再同步
- 让每轮改进都留下可追溯成文信息

## 相关文件

- `SKILL.md` — 技能定义与入口说明
- `INSTALL_FOR_AI.md` — 安装导向说明
- `VERSION` — 当前版本
- `CHANGELOG.md` — 版本变化记录
- `src/core/` — 核心实现（如 verifier、memory、engine）

## 备注

HeartFlow 的重点不再只是“更会做任务”，而是“更会用正确过程做正确的事”。

这次升级把 HeartFlow 从单次执行导向，推进到过程改进导向。
