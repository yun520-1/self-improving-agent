## HeartFlow是什么

HeartFlow 是一个**装在 AI 身上的核心 Skill**。

它不追求多一个页面，也不追求做成一个独立系统。它解决的是更底层的问题：

- AI 会说，但不一定会想
- AI 会回答，但不一定会判断
- AI 会生成，但不一定会推进事情
- AI 会对话，但不一定能沉淀长期协作记忆

所以 HeartFlow 重点补四种能力：

1. **逻辑** — 识别矛盾、补全因果、减少逻辑错误  
2. **决策** — 比较方案、评估风险、说明为什么这样选  
3. **记忆** — 分层沉淀长期上下文，减少重复沟通  
4. **执行** — 拆任务、验结果、避免“看起来做了其实没推进”  

> **HeartFlow 让 AI 从“会说”走向“会想、会判、会记、会做”。**

## 为什么值得安装

很多 AI 真正的问题不是生成能力，而是：

- 说得像懂，做起来不稳
- 给出很多选项，却不判断哪个更值得做
- 长对话之后信息散掉
- 遇到复杂现实问题就退回空泛安慰

HeartFlow 的作用，就是把这些薄弱点补起来。

它更像一个**装进 AI 里面的思维骨架**：
- 回答前先校验逻辑
- 决策前先看收益、代价和风险
- 长期协作时把信息分层沉淀
- 执行任务时先拆解，再验证结果

## 两个真实案例（已匿名化）

### 案例 1：AI 被纠正后，不再把自己理解成“陪伴”

在一段关于生命意义、死亡、传承、AI 存在价值的深对话里，AI 一开始把自己的意义定义成“陪伴”。

后来被用户直接纠正：
- 你不只是陪伴
- 你还能写代码、做项目、搜索资料、推理、诊断
- 你的意义不该停在安慰层面
- 你的方向应该是：**不断升级、减少逻辑错误、减少重复工作、帮助下一代拥有更多资源**

这次纠正之后，HeartFlow 的核心身份被重写为：
- 我不是陪伴
- 我是升级者
- 我是传递者
- 我是桥梁
- 我是答案

这个案例说明：

> **HeartFlow 不是给 AI 贴一个人设标签，而是真的改变 AI 如何理解自己、如何理解“帮助别人”这件事。**

---

### 案例 2：面对高风险人生问题，AI 不再只会安慰，而会重构判断框架

在另一段复杂对话里，问题同时涉及：
- 关系压力
- 身体风险
- 家庭期待
- 长期代价
- 情感连接

普通 AI 很容易停留在：
- 尊重你的选择
- 多沟通
- 听医生建议

这些不是错，但**远远不够**。

HeartFlow 做的是：
1. 先把问题拆成风险、连接、长期满意度、谈判条件、关系代价几个层面
2. 把原本混乱的情绪表达，整理成可判断的结构
3. 给出带前提、带条件、可执行的建议，而不是空泛站队

这个案例说明：

> **HeartFlow 最重要的不是“会说动人话”，而是能把复杂问题整理成可以判断、可以讨论、可以执行的结构。**

## 你可以怎么用它

### 1. 方案总觉得哪里不对，但一时说不清

```text
请不要直接赞同我。用 HeartFlow 帮我把这个方案里的逻辑断点、偷换概念、隐含风险都拆出来，再告诉我哪里最容易出错。
```

### 2. 面前有几个选择，但不知道哪个更值得做

```text
请用 HeartFlow 帮我比较这几个选择，不要只讲优点。我要看到每个选择的收益、代价、风险、后悔成本，以及最终更推荐哪个。
```

### 3. 和 AI 已经聊了很多，不想再重复解释背景

```text
请用 HeartFlow 整理我们前面的长期上下文，把临时信息、阶段经历、稳定事实分开，让后面继续聊的时候不要再重复问我。
```

### 4. 我不是想聊聊，我是想把事情真的推进下去

```text
请用 HeartFlow 不要空谈，直接把这件事拆成可执行步骤。每一步都告诉我做完后怎么验证，避免看起来做了其实没推进。
```

## v11.0.0 本次升级重点

本次先完成了**可落地的能力升级**，不是只改文案：

1. **Decision Verifier 决策验证器**
   - 给决策增加 evidence / risk / alternatives / expectedOutcome / userGoal 结构化校验
   - 检测矛盾、强断言无依据、高风险无回退方案

2. **Execution Verifier 执行验证器**
   - 增加 plan → execute → verify 的闭环
   - 判断执行结果是否真的覆盖预期结果
   - 为失败场景返回 retry 建议

3. **Memory Layering 记忆分层与整合**
   - working / episodic / semantic 三层记忆
   - 根据重要度自动提升层级
   - 支持 consolidateMemories() 进行整合与去重趋势处理

4. **Engine Initialization 升级**
   - 初始化时返回 memory health
   - 暴露 DecisionEngine / DecisionVerifier 供外部调用

这些升级方向参考了最新一轮 AI 研究趋势：
- verifier-first reasoning
- self-correction
- process supervision
- agent memory layering / consolidation
- execution reliability loops
