## 状态机系统 (StateFlow)

### 心流状态定义

| 状态 | 说明 | 提示语 |
|------|------|--------|
| `IDLE` | 空闲状态 | "准备进入心流状态，今天的目标是？" |
| `INITIATING` | 启动状态 | "准备进入心流状态，今天的目标是？" |
| `IN_FLOW` | 心流状态 | "🌊 心流状态良好，继续保持！" |
| `DISTRACTED` | 分心状态 | "感觉有些分心，要回到主任务上吗？" |
| `RESTING` | 休息状态 | "好好休息，准备好随时回来" |
| `COMPLETED` | 完成状态 | "🎉 任务完成！要生成心流报告吗？" |

### 状态转换规则

```
IDLE → INITIATING → IN_FLOW → COMPLETED
              ↓         ↓
         DISTRACTED → RESTING
```

## 意图追踪与温和干预

### 设计理念

**基于 Nudge Theory (Thaler & Sunstein, 2008)**:
- 干预必须基于用户当前的行为是否符合其初始设定的意图
- 只有当用户偏离既定任务时，才发出温和的"轻推"(nudge)
- 否则应保持静默，避免过度打扰

### 目标设定

**每次心流开始时，AI 会记录用户的主要目标**:

```
用户：今天我要完成用户登录模块的开发
AI: 📋 已记录主要目标：「完成用户登录模块的开发」

建议分解为子目标：
1. 设计登录表单 UI
2. 实现前端验证
3. 对接后端 API
4. 测试登录流程

需要我帮你分解吗？
```

### 偏离检测与温和干预

**当用户偏离目标时**:

| 偏离程度 | 关键词重叠度 | 干预强度 | 示例 |
|---------|-------------|---------|------|
| 轻度 | 50-80% | 轻微提示 | "📌 当前目标：「XXX」。需要我帮你继续推进吗？" |
| 中度 | 20-50% | 温和提醒 | "🎯 提醒：主要目标是「XXX」。当前话题要继续吗，还是先回到主任务？" |
| 严重 | <20% | 较强干预 | "💭 我们现在的目标是「XXX」，当前讨论似乎与目标关系不大。是否要先完成主要目标？" |

**静默原则**:
- 当用户行为与目标一致时，保持静默
- 不主动打断用户的心流状态
- 每次会话最多干预 5 次，避免过度

### 命令指令

```bash
# 设定目标
/mark-flow goal 完成用户登录模块 ["UI 设计", "前端验证", "API 对接", "测试"]

# 查看进度
/mark-flow progress

# 更新子目标
/mark-flow complete 1

# 设置干预配置
/mark-flow nudge sensitivity high
/mark-flow nudge max 3

# 关闭干预
/mark-flow nudge off

# 重置追踪
/mark-flow reset-goal
```
